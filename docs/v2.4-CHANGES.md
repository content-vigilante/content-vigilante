# v2.4 — agency & brand edition

## Per-workspace branding & white-label
- `Workspace` type gains `clientName`, `logoUrl`, `hourlyRate`, `currency`, `role`.
- New `WorkspacePanel` in Settings — switch workspace, edit branding inline, set role.
- `/share/[token]` reads `brand.name`, `brand.color`, `brand.logoUrl` from the snapshot
  and renders the portal with the client's logo + brand color (falls back to the owl).
- Exports' Share-portal generator now packs the active workspace's brand into the snapshot.

## Per-client billing rate
- Invoice generator hydrates from the active workspace's `hourlyRate`, `currency`, and
  `clientName`. Switch workspaces, the invoice numbers change automatically.
- All `€` literals in Exports replaced with the workspace's currency symbol.

## Approval workflow
- Post status enum extended: `idea` → `drafting` → `in-review` → `approved` → `scheduled` → `published`.
- Pipeline kanban now has 6 stages (was 4). Drag a post into `in-review`, click **Approve** on
  the card to advance it.
- `approvedBy` + `approvedAt` stamped on the post.
- Studio's Publish button is gated by workspace role:
  - `admin` / `reviewer` → see **Publish**.
  - `writer` → see **Submit for review** instead (creates the post in `in-review`).
- Reviewers see the **Approve** action; writers don't.

## Campaign tagging + rollups
- New `Campaign` type and seed data (`v2 Launch wave`, `Evergreen brand`).
- Posts and Leads gain optional `campaignId`. Seed data is pre-tagged.
- New `/dashboard/campaigns` page:
  - Per-campaign card with **posts × leads × pipeline × closed × net × ROI**.
  - Editable ad-spend field per campaign.
  - **Diagnose** button feeds the rollup into the LLM and returns a 5-bullet "what's working /
    what to try" read.
  - "How to use this" footer doubles as onboarding.

## CRM webhook (lead.hot)
- Moving a lead in the Pipeline now scores the new tier. If the move crosses into `hot`
  (score ≥ 70), the client posts `lead.hot` to `/api/notify/broadcast`, which fans out to
  every Slack / Discord / webhook destination you've configured.
- New `/api/notify/broadcast` route — used by the Pipeline today, available for any future
  trigger site (publish, watchlist diff, inbox add, etc.).

## Role-based UI gates (soft)
- `WorkspaceRole = 'admin' | 'writer' | 'reviewer'`.
- Studio: writers see "Submit for review" instead of Publish.
- Pipeline: writers don't see the **Approve** button.
- Documented as advisory until NextAuth + DB enforce it.
