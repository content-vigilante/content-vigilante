# v2.3 — Slack/webhook alerts + multimedia (image) audit

## Notifications (Slack / Discord / generic webhook)
- New `lib/notify.ts` — per-user notification target store keyed by sync token, KV-backed.
- New `POST/GET /api/notify/targets` — list, save, send test.
- Cron publish now broadcasts `post.published` and `post.publish_failed` events to every
  destination that opted into them.
- New **Notifications** card in Settings — add Slack/Discord/webhook destinations with a
  per-event opt-in matrix. One-click "Test" sends a real ping.

Future broadcast wiring (already supported in the helper, just needs the trigger points):
- `watchlist.changed` — fires when `/api/watchlist/check` detects a hash diff
- `inbox.new_comment` — fires on new inbox item
- `lead.hot` — fires when a `scoreLead` upgrade crosses the 70 threshold

## Multimedia (image) audit
- New `POST /api/audit-asset` — multipart image upload + optional brand-rule context.
- Vision LLM call: Anthropic Claude (Messages API, `image/base64`) or OpenAI Chat
  Completions (`image_url` data URL). Uses the user's own provider key.
- Returns structured JSON:
  ```json
  {
    "altText": "...",
    "description": "...",
    "ocrText": "...",
    "contextSummary": "...",
    "brandCompliance": {
      "score": 0-100,
      "palette": ["#hex", ...],
      "issues": [{ "kind": "color|logo|composition|text|accessibility", "message": "..." }]
    }
  }
  ```
- New `/dashboard/assets` page — drag-drop image, preview, score, palette, alt-text (with
  copy button), description + OCR, "Add to Context library" button.

Why this matters: a single creative now becomes (1) accessibility-ready (alt text),
(2) brand-checked, (3) searchable context that the post-idea suggester can use.
