# v2.2 — server-driven publishing, Meta integration, watchlist, dark social, NextAuth

## Server-driven cron publishing
- New `lib/platformTokens.ts` — per-user (sync-token-keyed) token storage on top of the existing
  `getSyncStore()` adapter. In-memory in dev, Vercel KV in prod.
- `POST /api/account/link` — copies the cookie-sealed OAuth tokens (LinkedIn, X, Meta, GA4)
  into KV under your sync token. Triggered from the new **AccountPanel** in Settings.
- `POST /api/account/queue` — push posts (`{ id, platform, text, scheduledFor }`) into your
  user's server-side queue. `GET` returns the queue + connected platforms.
- `/api/cron/publish` — now does real work. Iterates every linked user, picks up posts whose
  `scheduledFor` is in the past, publishes via the real platform APIs (LinkedIn `ugcPosts`,
  X `2/tweets`, Facebook `Page/feed`). Marks each as published. Returns a structured result.
- `vercel.json` cron stays at `*/15 * * * *` and is `CRON_SECRET`-gated.

## Facebook + Instagram (Meta Graph)
- `GET /api/meta/connect` — single OAuth flow that requests `pages_manage_posts` +
  `instagram_basic` + `instagram_content_publish`. Long-lived token exchange built in.
- `GET /api/meta/callback` — finishes OAuth, fetches the first Page + its IG business account,
  seals everything into `cv_fb` and (when an IG account exists) `cv_ig` cookies.
- `POST /api/facebook/publish` — posts to the Page feed.
- `POST /api/instagram/publish` — two-step Graph flow (`/media` → `/media_publish`). Requires
  a public `imageUrl` or `videoUrl` (Instagram text-only posts aren't supported by the API).
- `/api/connections/status` now reports `facebook` and `instagram` connections.

Env vars needed: `META_CLIENT_ID`, `META_CLIENT_SECRET`. Redirect URI:
`<origin>/api/meta/callback`.

## Competitor watchlist
- New `/dashboard/watchlist` page — add/remove competitor URLs.
- `POST /api/watchlist/check` — fetches each URL, strips scripts/styles/tags, hashes the
  resulting text. The UI tracks the previous hash per target and flags `changed: true` when
  the content shifts. Local-first (URL list lives in `localStorage`); rolling diff state is
  per-browser today.

## Dark social tracker
- `POST /api/track` — accepts `{ path, referrer, utm }` from any client. Classifies the channel
  (`search`, `social:<host>`, `referral:<host>`, `paid:<utm_source>`, `dark-social`) and stores
  a rolling window of 1,000 hits in the sync store.
- `GET /api/track` — channel totals + 20 most recent hits.
- New **Dark Social** card on `/dashboard/analytics`.
- Drop the one-line snippet shown in the empty state into your site's `<head>` to start tracking.

## NextAuth (optional)
- Minimal Google-provider scaffold via `next-auth@5.0.0-beta.31`.
- `/api/auth/[...nextauth]` route handler.
- Activates only when `AUTH_SECRET`, `AUTH_GOOGLE_ID`, `AUTH_GOOGLE_SECRET` are set on Vercel.
- The new **Account & token linking** card on Settings shows session state and the "Link now"
  button that pushes cookie tokens into KV.

## Activation summary (env vars on Vercel)

| Integration | Env vars |
|---|---|
| LinkedIn | `LINKEDIN_CLIENT_ID`, `LINKEDIN_CLIENT_SECRET` |
| X | `X_CLIENT_ID`, `X_CLIENT_SECRET` |
| Facebook + Instagram | `META_CLIENT_ID`, `META_CLIENT_SECRET` |
| GA4 | `GA4_CLIENT_ID`, `GA4_CLIENT_SECRET` |
| Cookie sealing | `CV_COOKIE_SECRET` |
| Vercel KV (sync + tokens + cron) | `KV_REST_API_URL`, `KV_REST_API_TOKEN` (auto on connect) |
| Cron auth | `CRON_SECRET` |
| NextAuth (optional) | `AUTH_SECRET`, `AUTH_GOOGLE_ID`, `AUTH_GOOGLE_SECRET` |
