# v1.2 — what's new

## Cross-device sync (KV-backed)
- Adapter pattern in `packages/web/lib/syncStore.ts`. Uses Vercel KV when
  `KV_REST_API_URL` + `KV_REST_API_TOKEN` are set, otherwise falls back to
  in-memory (good for dev / preview).
- New **Sync** card on `/dashboard/settings`:
  - Paste or generate a 16+ char opaque token (kept in `localStorage`).
  - **Push to cloud** — uploads posts/leads/workspaces/customGuide.
  - **Pull from cloud** — overwrites local snapshot.
  - Shows the adapter (KV vs. in-memory) so you know what backs the data.
- `/api/sync` now supports GET / POST / DELETE.

To enable real KV (3 min):
1. Vercel dashboard → Storage → Create KV.
2. Connect it to the `content-vigilante-vercel` project. Vercel auto-injects
   `KV_REST_API_URL` and `KV_REST_API_TOKEN`.
3. Redeploy.

## X (Twitter) publishing
- OAuth 2.0 with PKCE.
- New routes: `/api/x/connect`, `/api/x/callback`, `/api/x/publish`.
- Studio shows a **Publish** button when platform = X (uses `Send` icon).
- Connections card on Settings shows X status + @handle when connected.

Env vars: `X_CLIENT_ID`, `X_CLIENT_SECRET` (set on Vercel). Redirect URI:
`<origin>/api/x/callback`. Required scopes baked into `/api/x/connect`:
`tweet.read tweet.write users.read offline.access`.

## Scheduled publishing — first pass
- **Browser-driven (works today):** Calendar shows a **Publish N due** action
  for any scheduled posts whose `scheduledFor` is in the past on LinkedIn or X.
  Each publishes via the existing routes, then is marked `published`.
- **Server-driven (cron stub):** `vercel.json` adds a cron at
  `*/15 * * * *` hitting `/api/cron/publish`. The route is gated by
  `CRON_SECRET` and returns a stub today. To finish the server-side path we
  need to migrate platform tokens from per-browser cookies to per-user KV
  records — documented inline in the route source. Cleanly scheduled for v1.3.
