# v1.1 — Integrations & sync

What's wired in code and what you need to enable each piece on Vercel.

## PDF brand-guide ingestion
**Status:** fully working, no config.
Endpoint: `POST /api/ingest-guide` (multipart, field `file`).
UI: Guardrails page → "Upload your brand guide". Stored in `localStorage` (`cv:store:customGuide`) and used automatically by `/api/audit`.

## Google Analytics (GA4)
**Status:** code complete, needs Google Cloud OAuth client.

1. https://console.cloud.google.com → APIs & Services → Enable "Google Analytics Data API".
2. Create OAuth 2.0 Client ID (Web application). Authorized redirect URI:
   `https://content-vigilante-vercel.vercel.app/api/ga4/callback`
3. Vercel project env vars (Preview + Production):
   - `GA4_CLIENT_ID`
   - `GA4_CLIENT_SECRET`
   - `CV_COOKIE_SECRET` (any 32+ char random string)
4. Visit `/api/ga4/connect` once. Then on the Analytics page paste your property ID
   (format: `properties/123456789`) and click "Pull GA4".

Routes:
- `GET  /api/ga4/connect`   — starts OAuth.
- `GET  /api/ga4/callback`  — finishes OAuth, stores encrypted token in cookie `cv_ga4`.
- `POST /api/ga4/report`    — `{ propertyId, dateRange }` → returns Data API runReport.

## LinkedIn publishing
**Status:** code complete, needs LinkedIn Dev app.

1. https://www.linkedin.com/developers/apps → create app.
2. Auth → Authorized redirect URLs:
   `https://content-vigilante-vercel.vercel.app/api/linkedin/callback`
3. Products → request "Share on LinkedIn" + "Sign In with LinkedIn using OpenID Connect".
4. Vercel env vars:
   - `LINKEDIN_CLIENT_ID`
   - `LINKEDIN_CLIENT_SECRET`
   - `CV_COOKIE_SECRET` (same as GA4)
5. Visit `/api/linkedin/connect` once. Then on Studio with platform=LinkedIn, click **Publish**.

Routes:
- `GET  /api/linkedin/connect`   — starts OAuth.
- `GET  /api/linkedin/callback`  — token + userinfo, sealed in cookie `cv_li`.
- `POST /api/linkedin/publish`   — `{ text, visibility }` → posts to `/v2/ugcPosts`.

Scheduling (future): wire `vercel.json` cron → reads scheduled posts from DB → calls `/api/linkedin/publish`.

## Cross-device sync
**Status:** in-memory dev adapter shipped; swap to KV/Postgres for prod.

Contract:
- `GET  /api/sync` with header `x-cv-sync-token: <opaque 16+ char>` → returns `{ data, updatedAt }`.
- `POST /api/sync` with same header + body `{ data }` → persists.

To make it real:
- **Vercel KV (fastest):** `bun add @vercel/kv`, replace the `memory` Map in `packages/web/app/api/sync/route.ts` with KV `get`/`set`. Set `KV_REST_API_URL` + `KV_REST_API_TOKEN` in Vercel.
- **Postgres (more control):** add Drizzle or Prisma, single table `sync_blobs(token text primary key, data jsonb, updated_at timestamptz)`.

## Full auth (NextAuth)
**Status:** not built — sync token model gets you 80% with zero auth UX overhead.

When you want named accounts:
1. `bun add next-auth@beta @auth/drizzle-adapter`
2. Add `[...nextauth]/route.ts` with a Google or Email provider.
3. Replace the sync token header with `session.userId`.
4. Migrate the in-memory sync map to a `users` + `sync_blobs` schema.
